# s0c

`s0c` — CLI-инструмент для работы с проектами и базами данных платформы sb0rka.
Он помогает инженерам и AI-агентам быстро и надежно запускать нужные сценарии, получать доступ к API и psql.

Русский | [English] (README_EN.md)

## Установка и запуск

```bash
# Собрать локально:
go build -o s0c .

# Аутентифицироваться:
./s0c auth

# Открыть psql:
s0c psql
```

- Состояние авторизации сохраняется в `~/.s0c/auth.json`.
- Настройки по умолчанию сохраняются в `~/.s0c/config.json`.

## API-команды

- `s0c api plan` — показать текущий тарифный план пользователя
- `s0c api projects` — вывести список проектов
- `s0c api dbs` — вывести список баз данных (использует `--project-id` или настроенное значение по умолчанию)
- `s0c api dbs uri` — вернуть URI подключения к базе данных (использует настройки по умолчанию, если не переопределены)
- `s0c api dbs create --name <name> [--description <description>]` — создать базу данных в проекте
- `s0c psql` — открыть полноценную локальную интерактивную сессию `psql`, используя данные подключения sb0rka

Команды:

```bash
s0c api projects --pretty
s0c api dbs create --project-id <project-id> --name <database-name> --description "analytics cluster"
s0c psql --project-id <project-id> --database-id <database-id>
```

## Поведение, предсказуемое для AI-агентов

- при ошибке команда завершается с ненулевым кодом
- ошибки выводятся в `stderr` с префиксом `s0c:`
- JSON-ответы API печатаются в `stdout` — по умолчанию в одну строку, а с `--pretty` в читаемом формате
- команда `s0c api dbs uri` выводит в `stdout` только URI подключения

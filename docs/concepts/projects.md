Проект — это корневой контейнер для всей вашей работы в Sb0rka. Когда вы создаёте базу данных или сохраняете секрет, вы делаете это внутри конкретного проекта. Проекты изолированы друг от друга, поэтому ресурсы из одного проекта никогда не видны и не доступны из другого проекта.

Каждая создаваемая вами база данных и секрет связаны ровно с одним проектом через поле `project_id`. Ресурсы из одного проекта не могут быть использованы или доступны из другого. Вы можете создавать несколько проектов, чтобы разделять окружения (например, production и staging), команды или приложения.

## Поля проекта

| Поле          | Тип             | Описание                                                    |
| ------------- | --------------- | ----------------------------------------------------------- |
| `id`          | hex-string      | Уникальный идентификатор проекта.                           |
| `user_id`     | string (UUID)   | Идентификатор пользователя, владеющего проектом.            |
| `name`        | string          | Человекочитаемое имя.                                       |
| `description` | string \| null  | Необязательное описание проекта.                            |
| `is_active`   | boolean         | Активен ли проект или находится в очереди на удаление.      |
| `created_at`  | timestamp       | Дата и время создания проекта.                              |
| `updated_at`  | timestamp       | Дата и время последнего обновления проекта.                 |

## Лимиты проектов

Ваш тарифный план определяет, сколько проектов вы можете создать одновременно. Если вы достигли лимита `project_limit` в вашем плане, API вернёт ответ `403 Forbidden` при попытке создать новый проект. Обновите тарифный план, чтобы увеличить лимит, или запустите процесс удаления существующего проекта.
